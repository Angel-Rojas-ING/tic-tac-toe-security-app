6. Comando de Ejecución en Docker

Comando Básico de Ejecución

docker run -d --name tic-tac-toe-app -p 3000:3000 angel697/tic-tac-toe-security-app:latest

Comando Completo con Mapeo de Puertos y Volúmenes Persistentes

# Crear volumen para persistencia de datos docker volume create tic-tac-toe-data # Ejecutar contenedor con configuración completa docker run -d \ --name tic-tac-toe-production \ -p 3000:3000 \ --restart unless-stopped \ -v tic-tac-toe-data:/app \ --health-interval=30s \ --health-timeout=3s \ --health-retries=3 \ --log-driver json-file \ --log-opt max-size=10m \ --log-opt max-file=3 \ angel697/tic-tac-toe-security-app:latest

Verificación de Funcionamiento

# Verificar estado del contenedor docker ps | grep tic-tac-toe # Verificar logs docker logs tic-tac-toe-production # Verificar conectividad curl http://localhost:3000/api/session # Acceso web directo # http://localhost:3000

Configuraciones Adicionales

Mapeo de Puertos

  • -p 3000:3000: Puerto principal de la aplicación web
  • Alternativo: -p 8080:3000 (puerto personalizado)

Volúmenes Persistentes

  • -v tic-tac-toe-data:/app: Persiste base de datos SQLite
  • Ubicación: /app/database.sqlite dentro del contenedor

Opciones de Seguridad

  • --restart unless-stopped: Reinicio automático
  • Health checks: Monitoreo automático de salud
  • Log rotation: Gestión de logs con límites
